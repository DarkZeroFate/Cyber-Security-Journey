# 🛡️ Cyber Security Journey
*Transitioning from Business Management to Cybersecurity*

Hi! I'm 24, and I'm currently on a journey to pivot from a Business Management background into the world of Cybersecurity. 

I believe that the best way to learn is by "doing." Because I’m balancing a demanding full-time work schedule, I’m using this repository to document my progress, my experiments, and the technical hurdles I face along the way.

> **Note:** This isn't a repository of perfect, "textbook" tutorials. It’s my personal scratchpad—a place where I document my home lab, capture technical notes, and work through real-world problems.

---

## 🚀 Why This Repository?
I realized early on that reading theory is one thing, but actually implementing it is entirely different. I’m building this to prove to myself—and future hiring teams—that I can take a concept, build the environment, break it, and fix it.

I'm documenting my process because **troubleshooting is where the real learning happens.**

---

## 📂 Project Structure

Here’s how I’ve organized my notes and experiments:

- **[Network Lab](./network-lab/):** My home infrastructure setup (Kali Linux & Ubuntu).
    - *Includes:* VM setup, networking configurations, and my "Troubleshooting Log."
- **[Projects](./projects/):** Hands-on experiments.
    - *Includes:* Malware traffic analysis with Wireshark, Reverse TCP payload testing, etc.
- **[Notes](./notes/):** Random technical snippets I've picked up along the way.
    - *Includes:* Splunk, Sysmon, and other tools I'm exploring.

## 🛠️ My "Learn-by-Doing" Philosophy
I'm documenting everything from scratch. If you're also starting your journey, feel free to look around. You'll see that I'm learning by failing, so expect things to be a bit "work-in-progress." 

**I’m always open to feedback.** If you see a configuration I've messed up or a security practice I could improve, please feel free to reach out. I’m here to learn and grow.
